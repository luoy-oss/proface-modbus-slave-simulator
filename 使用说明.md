## 软件概述

### 软件简介
Modbus TCP从机调试软件是一个用于工业自动化领域的调试工具，它可以模拟Modbus TCP从机设备，接收来自PLC、HMI、SCADA系统等Modbus主机的读写请求。软件提供了完整的Modbus协议支持、实时通信监控、数据管理和模拟功能。

### 适用场景
1. **PLC程序调试**：模拟现场设备，测试PLC程序逻辑
2. **HMI画面测试**：为HMI提供模拟数据源，测试画面显示
3. **SCADA系统集成**：作为临时从机测试数据采集功能
4. **协议学习**：学习Modbus TCP协议工作原理
5. **设备测试**：测试Modbus主机的通信功能

## 功能详解

### 1. Modbus TCP服务器功能

#### 基本配置
- **IP地址配置**：
  - 输入本机IP地址（如192.168.1.113）
  - 使用0.0.0.0监听所有网络接口
  - 支持IPv4地址格式

- **端口配置**：
  - 默认端口：502（标准Modbus TCP端口）
  - 端口范围：1-65535
  - 避免使用系统保留端口

- **Unit ID配置**：
  - 默认值：1
  - 范围：0-255
  - 特殊值：0表示忽略Unit ID检查（响应所有请求）

#### 服务器控制
- **启动服务器**：
  1. 配置好IP、端口、Unit ID
  2. 选择需要的功能码
  3. 设置地址映射和字节序
  4. 点击"启动服务器"按钮

- **停止服务器**：
  1. 点击"停止服务器"按钮
  2. 等待状态显示"服务器已停止"
  3. 可以修改配置后重新启动

- **服务器状态**：
  - 未运行：服务器未启动
  - 运行中：服务器正常监听
  - 错误状态：显示具体错误信息

### 2. 通信监控功能

#### 通信日志
通信日志实时显示所有Modbus通信数据，包括：

- **时间戳**：请求到达时间（精确到毫秒）
- **客户端**：客户端IP地址和端口
- **事务ID**：Modbus事务标识符
- **Unit ID**：从机单元ID
- **功能码**：Modbus功能码及名称
- **地址**：操作的数据地址
- **数量**：操作的数据数量
- **数值**：写入的数值（写操作时显示）
- **原始数据**：十六进制格式的原始报文

#### 日志操作
- **自动滚动**：新日志自动显示在底部
- **手动查看**：使用滚动条查看历史日志
- **清空日志**：点击"清空日志"按钮清除所有记录
- **日志过滤**：通过功能码、地址等筛选关注的信息

### 3. 数据监控功能

#### 数据分类
软件支持四种Modbus数据类型的监控：

1. **线圈状态（Coils）**
   - 数据类型：布尔值（0/1）
   - 地址范围：0-65535
   - 功能码：01（读）、05（写单个）、15（写多个）

2. **离散输入（Discrete Inputs）**
   - 数据类型：布尔值（0/1）
   - 地址范围：0-65535
   - 功能码：02（读）

3. **输入寄存器（Input Registers）**
   - 数据类型：16位整数（0-65535）
   - 地址范围：0-65535
   - 功能码：04（读）

4. **保持寄存器（Holding Registers）**
   - 数据类型：16位整数（0-65535）
   - 地址范围：0-65535
   - 功能码：03（读）、06（写单个）、16（写多个）

#### 数据查看
- **表格显示**：以表格形式显示地址、状态、值
- **实时更新**：数据变化时自动更新显示
- **地址显示**：根据配置显示实际地址或偏移地址
- **值格式**：十进制显示，支持0-65535范围

#### 数据修改
- **双击修改**：双击表格中的数据项进行修改
- **手动设置**：使用地址和值输入框进行设置
- **批量修改**：支持多个地址的批量设置
- **修改确认**：修改后立即生效并更新显示

### 4. 数据模拟功能

#### 模拟任务类型
软件提供四种数据模拟任务：

1. **自增任务**
   - 功能：寄存器值自动递增
   - 配置：数据类型、起始地址、数量、起始值、最大值、步长、间隔
   - 应用：模拟计数器、累加器等设备

2. **位翻转任务**
   - 功能：线圈状态自动翻转
   - 配置：数据类型、起始地址、数量、间隔
   - 应用：模拟开关、指示灯等设备

3. **时间数据任务**
   - 功能：自动生成当前时间数据
   - 配置：基础地址、时间格式（时/分/秒或时/分/秒/毫秒）
   - 应用：模拟时钟设备、时间同步

4. **日期数据任务**
   - 功能：自动生成当前日期数据
   - 配置：基础地址、日期格式（年/月/日或年/月/日/星期）
   - 应用：模拟日历设备、日期记录

#### 任务管理
- **添加任务**：填写任务参数，点击"添加任务"按钮
- **任务列表**：显示当前所有运行中的任务
- **任务状态**：显示任务运行状态和参数
- **移除任务**：选择任务后点击"移除任务"按钮
- **任务控制**：所有任务自动运行，无需手动控制

#### 模拟参数说明
- **间隔时间**：任务执行间隔（毫秒），影响数据变化频率
- **地址范围**：确保地址在有效范围内（0-65535）
- **数值范围**：确保数值在有效范围内（0-65535）
- **数据类型**：选择正确的数据类型（线圈或寄存器）

## 配置说明

### 功能码配置
Modbus功能码是协议的核心，软件支持以下功能码：

| 功能码 | 名称 | 描述 | 默认启用 |
|--------|------|------|----------|
| 01 | 读线圈 | 读取线圈状态 | 是 |
| 02 | 读离散输入 | 读取离散输入状态 | 是 |
| 03 | 读保持寄存器 | 读取保持寄存器值 | 是 |
| 04 | 读输入寄存器 | 读取输入寄存器值 | 是 |
| 05 | 写单个线圈 | 写入单个线圈状态 | 是 |
| 06 | 写单个寄存器 | 写入单个寄存器值 | 是 |
| 15 | 写多个线圈 | 写入多个线圈状态 | 是 |
| 16 | 写多个寄存器 | 写入多个寄存器值 | 是 |

**配置建议**：
- 测试时建议启用所有功能码
- 生产测试时根据需要选择性启用
- 禁用不使用的功能码可以提高安全性

### 地址映射配置
不同设备对Modbus地址的理解不同，软件支持两种模式：

#### 1. 标准Modbus模式
- 地址从0开始
- 地址0对应第一个数据项
- 大多数设备使用此模式

#### 2. Pro-face模式
- 地址从1开始
- 地址1对应第一个数据项
- Pro-face等特定设备使用此模式

**选择依据**：
- 查看设备文档确定地址模式
- 测试时如果读取不到数据，尝试切换模式
- 大多数国产设备使用标准模式

### 字节序配置
字节序影响多字节数据的解析方式：

#### 1. 大端模式（Big Endian）
- 高位字节在前，低位字节在后
- 也称为"高位低字"
- 大多数Modbus设备使用此模式

#### 2. 小端模式（Little Endian）
- 低位字节在前，高位字节在后
- 也称为"低位高字"
- 部分特定设备使用此模式

**选择依据**：
- 查看设备文档确定字节序
- 测试时如果数据解析错误，尝试切换模式
- 不确定时使用大端模式

### 配置文件
软件会自动保存配置到`modbus_config.json`文件：

```json
{
  "ip": "192.168.1.113",
  "port": 502,
  "unit_id": 1,
  "enabled_functions": [1, 2, 3, 4, 5, 6, 15, 16],
  "address_offset": 0,
  "byte_order": "big"
}
```

**配置文件位置**：与主程序同一目录
**配置文件作用**：保存用户配置，下次启动时自动加载
**配置文件备份**：建议定期备份重要配置

## 使用示例

### 示例1：基本连接测试

#### 测试目标
验证Modbus主机能够正常连接和通信

#### 测试步骤
1. **启动软件**：
   - 运行`modbus_slave_debugger.py`或`ModbusSlaveDebugger.exe`
   - 软件界面正常显示

2. **基本配置**：
   - IP地址：192.168.1.113（或本机实际IP）
   - 端口：502

   - 在Modbus主机上配置相同的IP、端口、Unit ID

5. **验证通信**：
   - 查看通信日志是否有请求记录
   - 确认数据能够正常读写
   - 检查是否有错误信息

#### 预期结果
- 主机能够成功连接
- 通信日志显示正常的请求和响应
- 数据读写功能正常

## 故障排除

### 常见问题及解决方法

#### 问题1：无法启动服务器
**可能原因**：
1. 端口被占用
2. IP地址无效
3. 权限不足

**解决方法**：
1. 检查端口是否被其他程序占用
2. 确认IP地址格式正确
3. 以管理员权限运行软件
4. 尝试更换端口号

#### 问题2：客户端连接不上
**可能原因**：
1. 网络不通
2. 防火墙阻止
3. IP地址错误

**解决方法**：
1. 使用ping命令测试网络连通性
2. 检查防火墙设置，允许502端口
3. 确认客户端使用正确的IP和端口
4. 尝试使用0.0.0.0监听所有接口

#### 问题3：数据读取错误
**可能原因**：
1. Unit ID不匹配
2. 功能码未启用
3. 地址映射错误
4. 字节序错误

**解决方法**：
1. 确认Unit ID设置一致
2. 检查功能码是否启用
3. 尝试切换地址映射模式
4. 尝试切换字节序模式
5. 查看通信日志分析具体错误

#### 问题4：软件运行缓慢
**可能原因**：
1. 数据模拟任务过多
2. 通信日志过大
3. 系统资源不足

**解决方法**：
1. 减少数据模拟任务数量
2. 定期清空通信日志
3. 关闭不必要的监控项
4. 增加模拟任务间隔时间
5. 重启软件释放资源

#### 问题5：数据修改不生效
**可能原因**：
1. 数据类型错误
2. 地址超出范围
3. 值超出范围
4. 修改后未更新显示

**解决方法**：
1. 确认选择正确的数据类型
2. 检查地址是否在0-65535范围内
3. 检查值是否在有效范围内
4. 刷新数据监控页面
5. 查看通信日志确认修改操作

### 错误代码说明
软件可能返回以下Modbus错误代码：

| 错误码 | 名称 | 描述 | 可能原因 |
|--------|------|------|----------|
| 0x01 | 非法功能 | 功能码不支持或未启用 | 功能码配置错误 |
| 0x02 | 非法数据地址 | 地址无效或超出范围 | 地址配置错误 |
| 0x03 | 非法数据值 | 数据值无效或超出范围 | 数据值错误 |
| 0x04 | 从机设备故障 | 服务器内部错误 | 软件故障 |
| 0x0B | 网关路径不可用 | Unit ID不匹配 | Unit ID配置错误 |

### 调试技巧
1. **逐步测试**：从简单配置开始，逐步增加复杂度
2. **日志分析**：仔细查看通信日志，分析报文内容
3. **对比验证**：使用其他Modbus工具对比测试
4. **隔离测试**：排除网络、防火墙等外部因素
5. **参数调整**：尝试不同的配置组合

## 注意事项

### 安全注意事项
1. **测试环境使用**：本软件仅用于测试环境，请勿用于生产环境
2. **网络安全**：在公共网络中使用时注意网络安全
3. **数据安全**：不要使用敏感数据作为测试数据
4. **权限管理**：合理设置访问权限，避免未授权访问

### 性能注意事项
1. **资源占用**：大量数据模拟可能占用较多系统资源
2. **响应时间**：复杂配置可能影响GUI响应速度
3. **内存管理**：长期运行建议定期重启释放内存
4. **日志管理**：通信日志过大会影响性能，建议定期清理

### 使用注意事项
